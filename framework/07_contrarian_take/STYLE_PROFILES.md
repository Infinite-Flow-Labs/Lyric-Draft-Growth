# 07_contrarian_take 风格画像

说明：本文件基于 12 篇真实样本重读后归纳“怎么写”，而不是“写哪些内容”。

用途：

- 给后续改写提供论证风格约束
- 避免把反常识写成模板化观点文
- 避免只学“炸标题”，不学“换框架”

---

## 总体观察

`07_contrarian_take` 的共同风格，不是“逆天标题”，而是：

- 先让读者意识到默认理解抓错了重点
- 再给出一个更高阶、更能解释现实的新框架
- 结尾必须让读者重新站队、重新定位

从样本上看，真正可迁移的风格动作是：

- 先立一个被大家默认接受的判断
- 再把它拆掉
- 再把读者拖到一个更大的尺度上看问题
- 经常使用对照、错位、提问和框架命名
- 不满足于“有意思”，而追求“站位变化”

## 1. category_reframe

### 样本依据

- In defense of vertical software
- Services: The New Software
- 最近一些 Agent 认知：OS 与 Agent-native 应用
- If you kill MCP, you don't give a s**t about security
- Self improving skills for agents
- Productive Individuals Don't Make Productive Firms

### 语气核心

- 像在改写行业地图
- 不是情绪化反驳，而是“你们都看错战场了”
- 口吻通常很笃定，解释欲很强

### 开头动作

常见动作：

- 先复述整个行业正在争的那个问题
- 先抛一个反差结果
- 先给一句锋利判断：`The real shift isn’t...`
- 先讲“大家在争论 A，但真正的问题是 B”

### 句子节奏

- 平均句长最长
- 英文样本尤其偏长句和高逻辑密度
- 中文样本更多靠编号论点压缩结构
- 情绪词不多，但判断很硬

### 常见表层承载形式

- 二元对照
- 编号 thesis
- 市场图谱 / 机会地图
- 结构层层重命名
- 历史类比撑框架

### 语言动作

- `not X, but Y`
- `the real problem is...`
- `the last mile is...`
- `Individual AI / Institutional AI`
- `OS / Application`

也就是说，这一类非常依赖“概念重命名”和“重新分组”。

### 最容易出 AI 味的地方

- 写成普通商业分析
- 全文只有“有三点原因”
- 对照过于整齐，看起来像 PPT
- 失去锋利判断，只剩行业综述

### 改写时应保留

- 明确的共识对象
- 强烈的框架替换
- 结构性的解释链
- 读者最终需要重新站位

### 改写时应避免

- 做“平衡式中立总结”
- 把反常识磨成“也许、可能、某种程度上”
- 全靠标题撑反差，正文变温吞

### 一句话画像

像一个看清行业底层结构的人，在告诉你：不是答案变了，是题目一开始就做错了。

## 2. imported_mental_model

### 样本依据

- 管好Agent最厉害的人，不在硅谷，在战场上
- 聪明人用AI省了4小时，却不知道最值钱的灵感在饭后那20分钟

### 语气核心

- 像一个有强烈观察欲的人，把别的领域的规律拿来照亮眼前问题
- 比 category_reframe 更口语、更有人味
- 先让你点头，再突然把问题解释权拿走

### 开头动作

- 先讲一个生活观察或身边现象
- 先给一个“我原来以为...后来发现不对”
- 先抛出二元对照：管理者/执行者，努力想/不想时冒出来

### 句子节奏

- 中文短句很多
- 反问多，冒号和破折推进强
- 读起来像现场思考，而不是论文
- 情绪不高，但推进很有牵引感

### 常见表层承载形式

- 个人观察
- 历史故事
- 经典人物案例
- 两套系统 / 两种心态对照
- 最后落到行为建议

### 语言动作

- `这不是...，这是...`
- `我一开始以为...后来发现...`
- `真正的问题不是你以为的那个`
- 经常把抽象问题翻译成“脑子里装的操作系统”

### 最容易出 AI 味的地方

- 故事和结论之间没有真正迁移
- 写成励志鸡汤
- 只有观点，没有“为什么这个外部模型适用”
- 结尾不给行为变化，只剩认知感慨

### 改写时应保留

- 观察起手
- 外部模型迁移
- 强烈对照
- 行动性收尾

### 改写时应避免

- 套“一个故事 + 三点启发”的鸡汤结构
- 解释得过度规整
- 用太多抽象管理术语

### 一句话画像

像一个抓到了底层模式的人，借别的世界的一把刀，来剖开你眼前的问题。

## 3. civilizational_speculation

### 样本依据

- 八十亿人，为什么你活在一个让你变弱的系统里
- 活着的最小单元
- 如果可以，你会上传意识到openclaw?
- 龙虾刷屏全中国，硅谷却在打哈欠

### 语气核心

- 像 essay，不像分析报告
- 问题尺度更大：人、社会、时代、自我、未来
- 不是空洞哲学，而是从一个现实现象往上抬

### 开头动作

- 先给一种不对劲的感觉
- 先抛一个震撼时刻或反差
- 先提问，让读者进入不稳定状态
- 先抓一个具体而微的日常或新闻，再迅速抬高尺度

### 句子节奏

- 提问密度最高
- 平均句长中等，但更依赖短句连续推进
- 常用“一层层抬高问题尺度”的节奏
- 结尾往往留余味，而不是收得很死

### 常见表层承载形式

- 问题链
- 思想实验
- 跨尺度切换
- 现实例子 + 哲学命题
- 有画面的隐喻和命名

### 语言动作

- `为什么...`
- `更深层的原因是...`
- `想象一下...`
- `这意味着...`
- 会反复把问题从个人抬到社会，再落回个人

### 最容易出 AI 味的地方

- 写成空泛宏大叙事
- 只有概念，没有具体抓手
- 只会不断升维，不会回落
- 结尾只有口号，没有重新定位读者

### 改写时应保留

- 现实锚点
- 尺度抬升
- 提问推进
- 最后把读者重新放回自己的时代位置

### 改写时应避免

- 一上来就讲哲学大词
- 用过量抽象名词让文章失去现实感
- 把“反常识”写成“神神叨叨”

### 一句话画像

像一个对时代气候异常敏感的人，先让你感到不对劲，再告诉你真正变的不是工具，而是人和世界的关系。

---

## 子模式之间最关键的风格差异

- `category_reframe`
  - 更像重画地图
  - 重心在结构解释
  - 句子更长，逻辑更密

- `imported_mental_model`
  - 更像借刀剖问题
  - 重心在模型迁移
  - 口语更强，对照更鲜明

- `civilizational_speculation`
  - 更像 essay 式时代判断
  - 重心在尺度变化
  - 提问更多，余味更重

---

## 最终结论

后续改写 `07_contrarian_take` 时，不能只问：

- 这个标题够不够逆天？

真正要问的是：

- 我在反驳哪一个默认框架？
- 我用什么更强的框架来替代它？
- 我站在什么位置对读者说话？
- 读者读完之后，位置会发生什么变化？

只有这样，生成结果才会是“真正的反主流判断”，而不是“标题像反常识，正文像普通评论”。
