如果一个人去年还自称“AI hater”，今年却用 Claude 把一门新编程语言 Rue 做出来，这不是态度转弯这么简单，而是整个软件工程世界观被迫重写。Steve Klabnik 站在这个转弯口上，身份又偏偏足够重：他做过 Rust 十年，写过那本 Rust 社区几乎人人都知道的书，还把自己的人生一部分押在“编译器和文档”这种最不容易被流量解释清楚的东西上。

这场对话真正有意思的，不是他忽然开始夸 AI，而是他怎么一步一步被现实说服。2025 年初，他还会把自己描述成一个 AI 批评者；到了 2026 年初，他已经在谈 agentic coding、验证、测试框架和 Rue 的编译器后端。转折点并不神秘：他先是在 ChatGPT 刚出来时当成新奇玩具，后来又在 VS Code 里装上 Klein，才第一次意识到，原来这些工具不是“会不会聊天”，而是“能不能真的帮你推进工作”。

他最在意的，也许不是 AI 本身，而是围绕 AI 的叙事实在太吵了。对他来说，最大的问题不是观点分歧，而是事实分歧：一个工具明明已经能搜索网页、生成代码、迭代修改，另一边却还在争论“这根本不算”。在他眼里，这类争论卡住了更重要的问题：如果连工具到底能做什么都说不清，行业又怎么讨论该怎么用、谁来负责、哪些流程要重写？

他之所以会从怀疑者变成实践者，甚至有点像被一个完全不在软件圈的人逼着改观。那个人是他妈妈的男朋友，一个并不写代码的人，却不断告诉他：这东西能帮他写报告、起草内容、把空白页变成可编辑的材料。Steve 说，真正击中他的，是对方压根不把电脑当神圣机器，而是当成一种“本来就不可靠、但能勉强完成任务”的工具。软件工程师习惯了把确定性当作默认值，普通人却早就生活在“结果大概率对，但未必完全对”的世界里。

这也解释了他为什么会突然对“epistemic humility”反复上头。对他来说，AI 不是把程序员的自尊打碎了，而是把程序员长久以来的职业幻觉照得发亮：我们太习惯把自己的工作方式当成唯一正确的方式，太容易把“我不喜欢”误说成“这东西没用”。他并不主张把 AI 神化，恰恰相反，他反复强调的是，软件工程师需要承认，很多自己奉为圭臬的实践，未必真有证据支撑。

这个判断在 Rue 上变得具体了。Steve 说，Rue 原本是他想重新捡回“做编译器”的愿望：他年轻时就喜欢编译器，但没打算把所有编码时间都耗在手敲样板上。现在，Claude 这样的工具让他第一次觉得，自己也许能把时间放在架构、评审和设计上，而不是把精力全浪费在输入代码上。Rue 不是为了抢 Rust 或 Go 的位置，也不是为了做一门“更聪明”的语言，而是卡在 Rust 和 Go 之间，试图找到一块被忽略的空地：比 Go 更能往下走一点，比 Rust 更愿意让开发体验先活下来。

但这个项目也暴露了 agentic coding 最现实的一面：它不是自动化的魔法，而是需要训练的技能。Steve 把 AI 编程类比成 Vim，学习曲线很陡，但不能因为难学就说它没用。第一版 Rue 之所以走得磕磕绊绊，不只是模型更弱，也是他自己还不够会用，PR 开太大、节奏失控、项目一度卡住。到了后来，他干脆推翻重来，从头开始写，反而顺了很多。

真正让第二轮 Rue 起飞的，不是更激进的模型，而是更严厉的验证。Steve 做了一个语言规范，再写了一个自定义测试框架 Klab，把规格和测试案例连起来，让 agent 能自己判断工作对不对。对他来说，这才是 agentic coding 的关键：不是让模型“尽量聪明”，而是给它一个能收敛到正确性的反馈回路。测试不只是保险丝，而是方向盘。

他在这里还顺手拆掉了一个软件行业里很老的神话：DRY 未必永远是金科玉律。和 AI 一起写代码之后，他对重复的容忍度变高了。过去他会把重复视为结构性错误，现在他更愿意先让代码跑起来，再晚一点收拾。原因不是他突然不在乎质量了，而是他更清楚地意识到，很多所谓“好代码”其实是为了人类协作而设计的；如果机器也参与协作，最优解未必还是原来那一套。

这也是为什么他对 pull request、代码评审、甚至团队协作流程都变得更怀疑但也更开放。作为现在做源代码协作工具的人，他当然知道 PR 有价值，但他也会问：这些流程是在为今天设计，还是在为去年的软件世界设计？如果 AI 真的能把开发速度推高，旧流程到底是在保护质量，还是在拖慢新的工作方式？他没有给答案，只是指出一个更刺耳的现实：他在 Rue 上一个圣诞节就合了 100 个 PR，而且每个都“看过”了，但那种“看过”到底算不算真正审查，他自己也明白其中水分。

这才是这场对话最重要的地方。Steve 不是在给 AI 辩护，他是在逼软件行业承认一件更难的事：我们可能正在进入一个“速度极诱人、质量极难定义”的阶段。前沿实验室还在招人，说明开发者不是立刻要被淘汰；但如果连最前沿的人都开始把 agent 当成能干活的合作者，那软件工程真正要改写的，恐怕不是某个语法或某个框架，而是“信任应该建在哪里”这件事。

Rue 现在还不是可以随便给普通人用的语言，甚至离“成熟”都还有一段路：它在字符串、数组和引用的处理上还有问题，他自己也承认最近没法继续推进。但这并不妨碍它成为一个更大的信号。Steve Klabnik 这次转身，提醒软件行业的不是“AI 很厉害”，而是“别急着把自己的经验当作永恒真理”。如果 agentic coding 真要改变软件开发，最先被淘汰的未必是代码，而可能是程序员那种过度自信的确定感。
