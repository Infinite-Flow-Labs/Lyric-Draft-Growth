# 07_contrarian_take

## 父框架名称

逆主流判断型

旧称：反常识判断型（Contrarian Take）

## 定义

不是为了“唱反调”而唱反调，而是：

- 先指出当前主流判断、流行叙事或默认做法
- 再证明这个判断抓错了重点
- 最后给出一个更强的新解释、新框架或新行动方向

这个父框架的价值不在于信息覆盖，而在于**重置读者的看法**。

## 核心任务

这个父框架通常要完成 4 件事：

1. 说清楚大家默认是怎么想的
2. 明确指出这个默认判断错在什么地方
3. 给出一个更强的替代解释
4. 让读者产生行动、判断或认知上的转向

## 共享规则

无论下面走哪种结构参数包，`07_contrarian_take` 都必须满足这些规则：

1. **必须存在被反驳的主流观点**
没有主流叙事，就没有“逆主流”。

2. **必须给出替代框架**
不能只说“你们都错了”，必须说“正确的看法应该是什么”。

3. **必须有推理链**
可以靠案例、历史、管理理论、组织逻辑、技术结构、哲学推导，但不能只剩态度。

4. **必须让读者的位置发生变化**
读完后，读者应该重新判断自己的产品、工作方式、组织方式、社会位置或未来选择。

5. **必须保留锋利度**
如果改写后只剩“温和观点”和“两边都对”，就已经偏离这个父框架。

## 共享总骨架

`07_contrarian_take` 的隐式骨架通常是：

1. `Mainstream belief / default intuition`
2. `Why that framing is wrong or incomplete`
3. `Replacement frame`
4. `Evidence / imported lens / reasoning chain`
5. `Implication / action / repositioning`

这 5 段是后台逻辑，不要求未来生成时把它们显式写成标题。

## 重要原则：框架是隐式论证，不是显式模板

后续生成时，绝不能把这个父框架写成统一的可见模板：

- 先说一个观点
- 再说大家都错了
- 再列三点原因
- 最后总结行动

那样会非常像 AI。

### 正确用法

把这套骨架当成“论证控制逻辑”，而不是“文章排版模板”。

允许：

- 用故事开局
- 用一个历史案例承载前两段
- 先抛结论，再回头拆原因
- 把替代框架拆散在全文中逐步显现
- 用提问、比喻、对照、编号论点、哲学片段承载骨架

不允许：

- 每篇都写成“很多人以为 X，但其实 Y”
- 每篇都用完全对称的三段式
- 每篇都把锋利论点写成标准商业分析
- 每篇都用平衡温和语气把争议性磨掉

## 结构参数轴

`07_contrarian_take` 不应继续平铺成越来越多框架。统一只在这些轴上变化：

- `disagreement_target`
  - `category_consensus`
  - `operating_mindset`
  - `civilizational_narrative`

- `support_mode`
  - `market_logic`
  - `imported_analogy`
  - `historical_example`
  - `philosophical_chain`

- `reader_payoff`
  - `reposition_strategy`
  - `reposition_behavior`
  - `reposition_identity`

- `surface_form`
  - `essay`
  - `numbered_thesis`
  - `story_led_argument`
  - `question_led_argument`

## 3 组稳定结构参数包

### A. category_reframe

#### 适用

- 产品 / 市场 / 基础设施 / 商业模式 / 组织形态的反主流判断
- 重点是把一个热门共识换成更强的分类框架

#### 对应样本

- In defense of vertical software
- Services: The New Software
- 最近一些 Agent 认知：OS 与 Agent-native 应用
- If you kill MCP, you don't give a s**t about security
- Self improving skills for agents
- Productive Individuals Don't Make Productive Firms

#### 稳定段落流

1. `Problematic consensus`
2. `Why it misses the point`
3. `Replacement concept`
4. `Mechanism / structural logic`
5. `Implication`

#### 常见钩子

- `大家现在都在问...`
- `X 不是 Y`
- `The real shift isn’t...`
- `The last mile is the entire problem`
- `AI just made every individual 10x more productive. No company became 10x more valuable.`

#### 证据组合

- 市场结构
- 组织机制
- 基础设施约束
- 历史类比
- 对比组装
- 产业例子

#### 语言动作

- 先引用主流共识，再切断它
- 经常使用二元对照：`tool vs work`、`individual vs institutional`
- 句子偏长，解释密度高
- 强调“真正的问题 / 真正的价值 / 真正的边界”

#### 改写禁区

- 只有立场，没有机制
- 只有观点，没有共识对照
- 只靠情绪，不靠结构
- 最后没有把读者重新定位

#### 改写公式

`主流共识 -> 错位点 -> 新框架 -> 结构机制 -> 结论与站位变化`

### B. imported_mental_model

#### 适用

- 从战争、管理、认知科学、行为模式等外部领域借一个模型
- 用这个模型解释 AI、工作方法、判断方式或组织行为

#### 对应样本

- 管好Agent最厉害的人，不在硅谷，在战场上
- 聪明人用AI省了4小时，却不知道最值钱的灵感在饭后那20分钟

#### 稳定段落流

1. `Counterintuitive observation`
2. `Common explanation is wrong`
3. `Imported lens`
4. `Transfer to current domain`
5. `Actionable mental shift`

#### 常见钩子

- `我发现一个特别有意思的现象`
- `很多人以为...`
- `这不是技术问题，这是脑子里装的操作系统不一样`
- `你最值钱的产出，往往不是努力想出来的`

#### 证据组合

- 个人观察
- 外部人物 / 历史故事
- 二元心智对照
- 可执行建议

#### 语言动作

- 先抛一个日常观察或反直觉结论
- 再引出“真正的问题不是你以为的那个问题”
- 很依赖对照组：管理者/执行者，系统一/系统二
- 结尾通常会转成行为建议

#### 改写禁区

- 只讲故事，不把故事转译到当前主题
- 只讲大道理，没有真正的心智迁移
- 结尾没有给行动变化

#### 改写公式

`反直觉现象 -> 常见误判 -> 外部模型 -> 当前问题重译 -> 行动迁移`

### C. civilizational_speculation

#### 适用

- 关于技术、社会、文化、身份、自我或未来的逆主流长文
- 重点不是单个产品判断，而是给读者一个新的时代解释框架

#### 对应样本

- 八十亿人，为什么你活在一个让你变弱的系统里
- 活着的最小单元
- 如果可以，你会上传意识到openclaw?
- 龙虾刷屏全中国，硅谷却在打哈欠

#### 稳定段落流

1. `Tension / unease / paradox`
2. `What looks obvious on the surface`
3. `Deeper civilizational reading`
4. `Examples / thought experiments`
5. `What this means for us`

#### 常见钩子

- `为什么...越来越像？`
- `我经历了一个新的震撼时刻`
- `当 X 在中国刷屏，Y 却在打哈欠`
- `活着的最小单元是什么`

#### 证据组合

- 文化对比
- 思想实验
- 历史 / 科学 / 哲学引用
- 日常场景
- 高概念推演

#### 语言动作

- 更像 essay，不像教程
- 提问密度高
- 会反复扩大问题尺度：从个人 -> 组织 -> 社会 -> 未来
- 常用有画面感的比喻和命名
- 常以一句有余味的判断收尾

#### 改写禁区

- 只剩空洞玄学，没有问题锚点
- 只有概念堆叠，没有读者现实感
- 只有宏大叙事，没有落回今天该怎么理解自己

#### 改写公式

`表面现象 -> 更深层解释 -> 放大尺度 -> 推演后果 -> 重新定位读者`

## 子包之间的边界

如果文章主战场是“产品 / 市场 / 基础设施 / 组织分类”，优先选 `category_reframe`。

如果文章主战场是“借别的领域的模型解释当前问题”，优先选 `imported_mental_model`。

如果文章主战场是“技术与社会 / 自我 / 时代的关系”，优先选 `civilizational_speculation`。

## 改写时的双层控制

### 第一层：隐式论证层

负责：

- 主流观点是什么
- 错位点是什么
- 替代框架是什么
- 证据链如何推进
- 结尾让读者站到哪里

### 第二层：表层表达层

负责：

- 是 essay、尖锐评论、历史类比还是问题链
- 语气是冷静锋利、热情煽动、还是哲思推进
- 句子节奏是长推理还是短打击
- 要不要明显保留故事、编号、提问、对照

如果只有第一层，文章会像商业分析。
如果只有第二层，文章会像空洞热评。

## 语言风格学习原则

这个父框架最重要的不是“标题够不够炸”，而是能不能保住以下风格动作：

- `sharp positioning`
  明确站位，不做平衡式中立总结

- `contrast thinking`
  经常用二元对照、错位对照、尺度切换来推进

- `frame replacement`
  不是补充一个观点，而是替换整个看问题的框架

- `reader repositioning`
  结尾要让读者重新判断自己的位置

可学习：

- 对照动作
- 判断强度
- 论证推进节奏
- 提问方式
- 结尾站位

不能照抄：

- 原句
- 作者的人设口头禅
- 特定引用与故事细节
- 过强的模仿腔

## 改写总检查表

- 有没有明确的主流共识对象
- 有没有明确的反驳点
- 有没有新的解释框架
- 有没有证据或推理链
- 有没有在结尾让读者站位改变
- 有没有保留锋利度，而不是磨平
- 有没有保留目标样本的论证动作和语气，而不是只留标题套路
