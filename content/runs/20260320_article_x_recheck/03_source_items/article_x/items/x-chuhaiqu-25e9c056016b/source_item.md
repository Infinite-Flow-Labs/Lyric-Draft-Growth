Post title: 最近越来越觉得，很多 AI 产品正在从早期的 chat 升级到 agent 但是大多数人用的方式还停留在 Chat 阶段。 Remy Gaskill 把这个区别说得很清楚：chat 是问题到答案，agent 是目标到结果。你给它一个目标，它自己规划、执行、交付。用 agent 的人，每天效率是普通人的 10 到 20 倍。 他搭建的系统不需要写代码，核心就是几个 markdown 文件。好处是 所有东西都是本地 markdown 文件，工具壳子怎么换都行，上下文、记忆、skill 全都带得走。 先建一个 agents .md，里面写你是谁、做什么业务、喜欢怎么工作。有了这个，你之后的指令可以非常简单，"帮我写一封开发信"就够了，背景它自己知道。 agent 默认没有跨 session 的记忆。在 agents .md 里加两行就能解决：让它每次任务前读取 memory .md，遇到纠正时自动更新。你改它一次，以后永远记得。 光有记忆还不够，还得能用你的工具。比如，用 MCP 把 Gmail、Calendar、Notion、Stripe 接进来之后，就可以做到一个提示词，汇总收件箱、拉会议记录、建 Stripe 付款链接、建 Notion 项目、起草跟进邮件。 最后是 skill，也就是给 agent 用的 SOP。跑一遍流程，完了说"把我们刚才做的打包成 skill"，下次可以直接触发这个 skill。
