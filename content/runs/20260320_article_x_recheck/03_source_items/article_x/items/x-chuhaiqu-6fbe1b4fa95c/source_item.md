Post title: Felix 有 7 年设计经验，先后在 Gotrade（YC S19）和 ADPList 工作。以前上线一个功能要写 brief、等排期、跟工程师来回沟通，几天起步。现在靠两个东西，流程完全变了。 第一个是 Figma MCP。 通过 MCP 把 Claude Code 连上 Figma，Claude 可以直接读组件、design token、间距。你指向一个 frame 告诉它要实现什么，它照着设计稿写代码。设计稿和最终产物第一次真正对齐了。 第二个是 Claude Code 的构建循环。 用自然语言描述功能，Claude 出第一版，你视觉上检查，告诉它哪不对，改，循环。 他说多年练出来的设计眼光在这个流程里反而是最大优势。 padding 不对、字重不对、交互不顺，一眼就能看出来，直接让 Claude 改。他在 ADPList 做的第一个内部工具，工程师估计两天，他四个小时搞定。 原型阶段变化也大。 以前 Figma 里做可点击原型要两小时，现在跟 Claude Code 描述交互直接生成能跑的代码，手机上就能测，30 分钟搞定。能跑的东西和看起来能跑的东西，拿到的反馈质量完全不一样。 还有个容易忽略的点是 context files。维护一份文件写清楚产品、用户、设计原则，每次 session 开头加载，产出质量和效率会比直接口喷高效太多太多。
